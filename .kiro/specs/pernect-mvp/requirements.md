# Requirements: Pernect MVP

## Overview
Pernect（パーネクト）のMVP（Minimum Viable Product）要件定義。
AI分析による自己発見プラットフォームの初期リリースに必要な機能を定義する。

## Non-Functional Requirements

### NFR-1: パフォーマンス
- アプリ起動時間: 3秒以内
- 画面遷移: 300ms以内
- API応答時間: 1秒以内
- オフライン時の基本操作対応

### NFR-2: セキュリティ
- JWT認証による安全なAPI通信
- 機密データの暗号化保存
- プライバシーポリシー準拠
- GDPR対応（将来）

### NFR-3: ユーザビリティ
- 直感的なUI/UX
- 日本語/英語対応
- アクセシビリティ対応
- レスポンシブデザイン（タブレット対応）

### NFR-4: 信頼性
- アップタイム: 99.5%以上
- データバックアップ
- エラーハンドリングとリカバリー

### NFR-5: スケーラビリティ
- 初期: 1,000ユーザー対応
- 成長期: 100,000ユーザー対応
- Convexの自動スケーリング活用

## MVP Scope Definition

### Phase 1: Core MVP（優先度: 高）
1. **FR-1**: ユーザー認証システム ✅
2. **FR-2.1**: MBTI診断テスト（基本版）
3. **FR-2.3**: 結果・履歴管理（基本版）

### Phase 2: Enhanced MVP（優先度: 中）
4. **FR-3**: AIチャット機能（基本版）
5. **FR-4.1**: 相性診断
6. **FR-5**: 通知システム完成

### Phase 3: Monetization（優先度: 低）
7. **FR-6**: サブスクリプション
8. **FR-4.2**: ユーザー検索・接続
9. **FR-3.2**: パーソナライズ分析（高度版）

## Acceptance Criteria

### AC-1: MBTI診断テスト完了条件
- [ ] ユーザーが診断を開始できる
- [ ] 全質問に回答できる
- [ ] 結果が正しく算出される
- [ ] 結果がプロフィールに反映される
- [ ] 履歴として保存される

### AC-2: AIチャット完了条件
- [ ] メッセージを送信できる
- [ ] 応答を受信できる
- [ ] 履歴が保存される
- [ ] 基本的なアドバイスが得られる

### AC-3: 相性診断完了条件
- [ ] 2つのMBTIタイプで相性スコアが算出される
- [ ] 詳細な相性説明が表示される
- [ ] 相性の良いタイプが推薦される

## Dependencies

### External Services
- **Clerk**: 認証サービス
- **Convex**: バックエンド/データベース
- **OpenAI API**: AIチャット（予定）
- **App Store Connect**: iOS課金
- **Google Play Console**: Android課金

### Internal Dependencies
- FR-2（診断）→ FR-3（AI分析の基盤データ）
- FR-2（診断）→ FR-4（相性診断の基盤データ）
- FR-1（認証）→ 全機能の前提条件

---

## Risks and Mitigations

| リスク | 影響度 | 対策 |
|--------|--------|------|
| AI API コスト増大 | 高 | 使用量制限、キャッシュ活用 |
| 診断精度への不満 | 中 | 科学的根拠の明示、免責事項 |
| プライバシー懸念 | 高 | 透明性のある説明、データ削除機能 |
| 競合サービスの台頭 | 中 | 差別化機能の継続開発 |
