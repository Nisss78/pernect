# Implementation Plan: Last Lover Test

最後の恋人診断（Last Lover Test）の実装タスク

---

## タスク一覧

- [x] 1. データベーススキーマの拡張
- [x] 1.1 16タイプのキャラクター情報を格納するテーブルを追加
  - タイプコード（ESFJ等）をキーとしてキャラクター名、説明、強み、弱みを保持
  - 恋愛スタイル、理想のデート、コミュニケーションスタイルのフィールドを追加
  - 絵文字フィールドで各タイプを視覚的に表現
  - タイプコードによる高速検索用のインデックスを設定
  - _Requirements: 3.4, 9.3_

- [x] 1.2 タイプ間の相性情報を格納するテーブルを追加
  - 基準タイプと相手タイプのペアで相性レベルを管理
  - 相性レベル（best/good/neutral/challenging）と理由を保持
  - タイプ単位および組み合わせ単位での検索インデックスを設定
  - _Requirements: 4.1, 4.2, 9.4_

- [x] 2. キャラクタータイプと相性のシードデータ作成
- [x] 2.1 16種類の恋愛キャラクタータイプを定義
  - ESFJ（献身的な世話焼きさん）、INFP（ロマンチックな夢見る人）など全16タイプ
  - 各タイプに固有のキャラクター名、絵文字、詳細説明を設定
  - 恋愛における強み3-4個、注意点2-3個を各タイプに設定
  - 恋愛スタイル、理想のデート、コミュニケーションスタイルの説明文を作成
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5_

- [x] 2.2 タイプ間の相性データを定義
  - 各タイプに対して相性の良いタイプを最大3つ設定
  - 各タイプに対して注意が必要なタイプを最大2つ設定
  - 相性の理由を簡潔な文章で各組み合わせに設定
  - 相性は非対称（AからBへの相性 ≠ BからAへの相性）を考慮
  - _Requirements: 4.1, 4.2, 4.3, 4.4_

- [x] 3. バックエンド機能の実装
- [x] 3.1 キャラクタータイプの取得機能を実装
  - タイプコードから対応するキャラクター情報を取得する機能
  - 全タイプの一覧を取得する機能
  - 認証不要で公開データとしてアクセス可能に設定
  - _Requirements: 3.4, 9.3_

- [x] 3.2 相性情報の取得機能を実装
  - タイプコードから対応する相性情報一覧を取得する機能
  - 相性レベルでソートして返却（best → good → challenging）
  - 相手タイプのキャラクター名も併せて取得
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 9.4_

- [x] 3.3 診断結果計算ロジックに恋愛診断の分析データを追加
  - 既存の診断結果計算機能に16タイプの分析データを追加
  - 各タイプのsummary、description、strengths、weaknesses、recommendationsを定義
  - dimension scoringパターンを活用した4軸スコア計算
  - _Requirements: 2.1, 7.1, 9.5_

- [x] 4. 診断テストのシードデータ作成
- [x] 4.1 診断テスト定義を作成
  - テスト名「最後の恋人診断」、説明文、カテゴリ「love」を設定
  - ピンク〜パープル系グラデーションカラー（#ec4899 → #a855f7）を設定
  - ハートアイコン、所要時間5分、質問数16問を設定
  - dimension scoringタイプを指定
  - _Requirements: 8.1, 8.2, 8.3, 9.1_

- [x] 4.2 16問のシナリオベース質問を作成
  - E/I軸（外向/内向）に対応する質問を4問作成
  - S/N軸（現実/直感）に対応する質問を4問作成
  - T/F軸（論理/感情）に対応する質問を4問作成
  - J/P軸（計画/柔軟）に対応する質問を4問作成
  - 各質問に恋愛テーマの絵文字を含める
  - 各質問に2択の選択肢と対応するスコアキーを設定
  - _Requirements: 1.1, 1.2, 1.3, 1.5, 6.2, 9.2_

- [x] 5. 結果画面の相性セクション実装
- [x] 5.1 相性の良いタイプの表示機能を実装
  - 診断結果タイプに基づいて相性の良いタイプを最大3つ表示
  - 各相性タイプのキャラクター名と絵文字を表示
  - 相性の理由を簡潔に説明するテキストを表示
  - ピンク〜パープル系のカード形式でデザイン
  - _Requirements: 4.1, 4.3, 4.4, 5.1_

- [x] 5.2 注意が必要なタイプの表示機能を実装
  - 診断結果タイプに基づいて注意が必要なタイプを最大2つ表示
  - 各タイプのキャラクター名と相性の理由を表示
  - 警告的ではなく建設的なアドバイス形式で表示
  - _Requirements: 4.2, 4.3, 4.4_

- [x] 6. 結果画面のビジュアル強化
- [x] 6.1 恋愛テーマのグラデーション背景を実装
  - ピンク〜パープル系のグラデーション背景を結果画面に適用
  - キャラクタータイプ表示エリアにも同系色のグラデーションを適用
  - 視認性を確保しつつ恋愛診断らしい雰囲気を演出
  - _Requirements: 5.1, 5.3_

- [x] 6.2 結果表示のアニメーションを実装
  - 結果確定時のフェードインアニメーションを追加
  - キャラクター名と絵文字の登場アニメーションを追加
  - 4軸スコアバーの伸びるアニメーションを追加
  - _Requirements: 5.2, 5.4_

- [x] 6.3 4軸スコアの視覚的表示を実装
  - E/I、S/N、T/F、J/Pの各軸スコアをバー形式で表示
  - 各軸の傾向（どちらが優勢か）を視覚的に分かりやすく表示
  - 恋愛テーマに合った配色でスコアバーをデザイン
  - _Requirements: 5.5_

- [x] 7. 質問画面のUI強化
- [x] 7.1 質問画面に恋愛テーマのデザインを適用
  - ピンク〜パープル系のグラデーション背景を適用
  - 選択肢をタップしやすいカード形式で表示
  - 選択時のアニメーションフィードバックを追加
  - _Requirements: 6.1, 6.3, 6.4_

- [x] 7.2 進捗表示の強化
  - 画面上部に進捗バーを表示
  - 現在の質問番号と総質問数を表示
  - 恋愛テーマに合った進捗バーのデザイン
  - _Requirements: 1.4, 6.5_

- [x] 8. パフォーマンスと信頼性の確保
- [x] 8.1 回答データのキャッシュ機能を実装
  - 診断中の回答を一時的にローカルにキャッシュ
  - アプリ中断時に進捗を保持
  - 次回起動時に中断した診断を再開可能に
  - _Requirements: 10.3, 10.4_

- [x] 8.2 エラーハンドリングの実装
  - ネットワークエラー時にエラーメッセージを表示
  - 再試行オプションを提供
  - 質問遷移が300ms以内で完了することを確認
  - _Requirements: 10.1, 10.2_

- [x] 9. テスト一覧への統合
- [x] 9.1 テスト一覧画面への表示統合
  - 最後の恋人診断をテスト一覧に表示
  - ハートアイコンとピンク〜パープル系グラデーションを適用
  - カードタップで診断画面に遷移
  - _Requirements: 8.1, 8.2, 8.3, 8.4_

- [x] 9.2 診断ステータスバッジの表示
  - 未開始/進行中/完了のステータスを表示
  - 進行中の場合は進捗状況も表示
  - _Requirements: 8.5_

- [x] 10. 統合テストと動作確認
- [x] 10.1 診断フロー全体の動作確認
  - テスト一覧からの診断開始フロー
  - 16問の質問回答フロー
  - 4軸スコア計算と16タイプ判定の正確性
  - 結果画面の表示（キャラクター、相性、スコア）
  - _Requirements: 全体_

- [x] 10.2 エッジケースの確認
  - 同点スコア時のタイプ決定
  - アプリ中断・再開時の進捗復元
  - ネットワークエラー時の挙動
  - 16タイプすべてが正しく表示されることの確認
  - _Requirements: 2.1-2.9, 10.1-10.4_
