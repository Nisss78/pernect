# MBTI性格診断（本格版60問）要件定義

## 概要

60問の質問でユーザーのMBTI性格タイプを判定する本格的な性格診断テスト。日常生活、仕事、対人関係のパターンから16種類の性格タイプに分類する。

## 機能要件

### FR-1: 診断テスト実行

**FR-1.1: テスト開始**
- ユーザーがテスト一覧から「MBTI性格診断」を選択して開始できる
- 開始前にテストの説明（60問、約15分）を表示する

**FR-1.2: 質問表示**
- 60問の質問を1問ずつ順番に表示する
- 各質問は2択の強制選択形式（A/B）で回答
- 進捗バー（例: 15/60）を表示する
- 質問の順序は4軸（E/I, S/N, T/F, J/P）が混在する形式

**FR-1.3: 回答保存**
- 各回答をリアルタイムで一時保存（中断対応）
- 診断を中断した場合、再開時に続きから回答可能

**FR-1.4: 結果計算**
- 60問の回答からスコアを集計（dimension scoring）
- 各軸で多い方の文字を組み合わせて結果タイプを決定
- 例: E=10, I=5, S=7, N=8, T=6, F=9, J=11, P=4 → ENFJ

### FR-2: 結果表示

**FR-2.1: 基本結果**
- 判定された16タイプのいずれか（例: ENFP）を表示
- タイプの日本語呼称（例: "広報運動家"）を表示
- 1行サマリー説明を表示

**FR-2.2: 詳細分析**
- タイプの詳細説明を表示
- 強み一覧を表示
- 弱み一覧を表示
- アドバイス/推奨事項を表示

**FR-2.3: スコア表示**
- 4軸それぞれのスコアバランスを表示
- 例: E 67% / I 33%

### FR-3: データ保存

**FR-3.1: 結果保存**
- 診断結果をtestResultsテーブルに保存
- usersテーブルのmbtiフィールドを更新

**FR-3.2: 履歴管理**
- 過去の診断結果を履歴として保持
- 最新の結果をプロフィールに反映

## 非機能要件

### NFR-1: パフォーマンス
- 質問切り替えは300ms以内で完了
- 結果計算は1秒以内で完了

### NFR-2: ユーザビリティ
- 質問選択後に自動で次の質問へ進む（300ms待機後）
- 戻るボタンで前の質問に戻れる
- 離脱率を下げるため15問ごとに進捗を強調表示

### NFR-3: 信頼性
- 回答の一時保存により中断しても再開可能
- ネットワークエラー時は適切なエラーメッセージを表示

## 受け入れ基準

### AC-1: テスト実行
- [ ] 60問すべて回答できる
- [ ] 質問順序が4軸混在形式になっている
- [ ] 進捗バーが正しく表示される
- [ ] 中断後に再開できる

### AC-2: 結果表示
- [ ] 16タイプのいずれかが正しく判定される
- [ ] タイプ名、サマリー、詳細説明が表示される
- [ ] 強み、弱み、アドバイスが表示される
- [ ] 4軸のスコアバランスが表示される

### AC-3: データ整合性
- [ ] testResultsに正しく保存される
- [ ] usersテーブルのmbtiフィールドが更新される
- [ ] 既存の診断（Last Lover, BIG5）に影響がない

## 技術仕様

### スコアリング
- **タイプ**: dimension
- **軸**: E/I, S/N, T/F, J/P
- **各軸の質問数**: 15問
- **計算方法**: calculateDimensionScores()を使用

### データモデル
- **tests**: slug="mbti", category="personality"
- **testQuestions**: 60件（order 1-60）
- **testResults**: resultType="XXXX"（16タイプ）

### UI
- **グラデーション**: #6366f1 → #8b5cf6（Indigo→Purple）
- **アイコン**: brain
