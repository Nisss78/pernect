# Requirements: 診断テストシステム

## Overview
汎用的な診断テストの基盤システム。
MBTI診断、エニアグラム診断、キャリア診断など、様々なタイプ診断を統一的に扱えるプラットフォーム。

---

## Core Concept

### 診断の流れ
```
テスト選択 → 質問に回答（5問程度）→ 結果算出 → 結果保存 → 履歴に蓄積
```

### 診断結果の活用
- プロフィールに表示
- 他の診断結果と組み合わせた分析
- 変数としてAIチャットなどで参照
- 相性診断の基盤データ

---

## Functional Requirements

### FR-1: 診断テスト定義
**Priority**: 🔴 High

#### FR-1.1: テストマスタ管理
- [ ] テストの基本情報（タイトル、説明、カテゴリ）
- [ ] テストの設定（質問数、所要時間、結果タイプ）
- [ ] テストの表示設定（アイコン、グラデーション色）
- [ ] 有効/無効の切り替え

#### FR-1.2: 質問管理
- [ ] 質問文の登録
- [ ] 選択肢の登録（通常は2〜5択）
- [ ] 各選択肢にスコア/タイプ加算を設定
- [ ] 質問の順序管理

#### FR-1.3: 結果タイプ定義
- [ ] 結果タイプの登録（例: ENFP, タイプ1, キャリアA）
- [ ] 結果の説明文
- [ ] 結果の詳細情報（強み、弱み、アドバイス）

---

### FR-2: 診断実行
**Priority**: 🔴 High

#### FR-2.1: 診断フロー
- [ ] テスト開始処理
- [ ] 質問の順次表示
- [ ] 選択肢から1つ選択
- [ ] 回答の一時保存
- [ ] 前の質問に戻る機能
- [ ] 中断・再開機能

#### FR-2.2: 結果算出
- [ ] 回答からスコア集計
- [ ] 結果タイプの決定（最高スコアのタイプ）
- [ ] パーセンテージ/スコア詳細の算出

---

### FR-3: 結果保存・管理
**Priority**: 🔴 High

#### FR-3.1: 結果保存
- [ ] 診断結果をデータベースに保存
- [ ] ユーザーIDとの紐付け
- [ ] 診断日時の記録
- [ ] スコア詳細の保存

#### FR-3.2: 履歴管理
- [ ] 過去の診断結果一覧表示
- [ ] テスト種類別のフィルタリング
- [ ] 結果詳細の再表示
- [ ] 同じテストの複数回受験対応

#### FR-3.3: 変数としての活用
- [ ] ユーザーの最新診断結果を取得するAPI
- [ ] 特定テストの結果を取得するAPI
- [ ] 結果を文字列/オブジェクトとして参照可能

---

### FR-4: UIコンポーネント
**Priority**: 🔴 High

#### FR-4.1: テスト一覧画面
- [ ] 利用可能なテスト一覧
- [ ] テストカード（アイコン、タイトル、所要時間）
- [ ] 進捗状態の表示（未受験/完了）

#### FR-4.2: 診断実行画面
- [ ] 質問カード表示
- [ ] 選択肢ボタン
- [ ] 進捗バー（例: 3/5）
- [ ] 戻る/次へナビゲーション

#### FR-4.3: 結果表示画面
- [ ] 結果タイプの大きな表示
- [ ] スコア/パーセンテージのグラフ
- [ ] 詳細説明テキスト
- [ ] プロフィールに保存ボタン

#### FR-4.4: 履歴画面
- [ ] 診断履歴リスト
- [ ] 日付、テスト名、結果タイプ表示
- [ ] タップで詳細表示

## Sample Tests (MVP)

### 1. MBTI診断（簡易版）
- **質問数**: 5問
- **結果タイプ**: 16種類（ENFP, INTJ, etc.）
- **スコア方式**: 4次元（E/I, S/N, T/F, J/P）

### 2. エニアグラム診断（簡易版）
- **質問数**: 5問
- **結果タイプ**: 9種類（タイプ1〜9）
- **スコア方式**: 9タイプ別スコア

### 3. キャリアタイプ診断
- **質問数**: 5問
- **結果タイプ**: 4種類（リーダー型, サポート型, クリエイター型, アナリスト型）
- **スコア方式**: 4タイプ別スコア

## Data Model Concept

### Test（テスト定義）
```
- id
- slug: "mbti-simple"
- title: "MBTI診断（簡易版）"
- description: "5問で分かるあなたの性格タイプ"
- questionCount: 5
- resultField: "mbti" // 保存先フィールド名
- scoringType: "dimension" | "single" // スコア方式
```

### Question（質問）
```
- id
- testId
- order: 1
- text: "あなたはパーティーで..."
- options: [
    { label: "多くの人と話す", scores: { EI: "E" } },
    { label: "少人数で深く話す", scores: { EI: "I" } }
  ]
```

### Result（結果）
```
- id
- userId
- testId
- resultType: "ENFP"
- scores: { E: 3, I: 2, S: 1, N: 4, ... }
- completedAt
```

## Acceptance Criteria

### AC-1: 診断完了
- [ ] ユーザーが5問の質問に回答できる
- [ ] 回答に基づいて結果タイプが算出される
- [ ] 結果がデータベースに保存される

### AC-2: 結果表示
- [ ] 結果タイプが画面に表示される
- [ ] スコア詳細が確認できる
- [ ] 説明文が読める

### AC-3: 履歴確認
- [ ] 過去の診断結果一覧が見られる
- [ ] 各結果の詳細が確認できる

### AC-4: 変数活用
- [ ] `user.latestResults.mbti` のように結果を参照できる
- [ ] AIチャットなどで診断結果を文脈として使える

---

## Out of Scope (MVP)
- 複雑な算出アルゴリズム（シンプルなスコア加算のみ）
- 質問のランダム化
- 回答の分析・統計機能
- 結果のシェア機能
- 有料テスト
